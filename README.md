# FinTablo — Диагностика роста практики финансиста

## Запуск
1. Открой `index.html` в браузере (Chrome, Firefox, Safari)
2. Работает локально (file://) — без сервера и внешних зависимостей

## Структура
```
index.html       — квиз (10 вопросов + промежуточный результат + форма захвата)
report.html      — персональный отчёт (полный / превью)
thankyou.html    — страница после отправки контакта
css/style.css    — дизайн-система
js/quiz.js       — логика квиза, навигация, инсайты, Яндекс.Метрика
js/calculator.js — формулы и расчёты метрик
js/report.js     — генерация персонализированного отчёта
js/email.js      — обработка лида (localStorage, подготовка к AmoCRM)
js/storage.js    — localStorage + сбор UTM-меток
```

## Аналитика
Встроена Яндекс.Метрика (счётчик 61131877). Цели:
- `quiz_started` — начал квиз
- `quiz_step_N` — прошёл шаг N (1-10)
- `quiz_intermediate` — дошёл до промежуточного результата
- `quiz_capture_screen` — дошёл до формы захвата
- `lead_submitted` — отправил контакт
- `report_preview_nocontact` — запросил превью без контакта
- `report_preview_viewed` / `report_full_viewed` — просмотр отчёта
- `report_cta_clicked` — нажал CTA в отчёте
- `thankyou_viewed` — просмотр страницы «спасибо»

### Настройка целей в Яндекс.Метрике
1. Откройте Метрику → Настройка → Цели → Добавить цель
2. Для каждой цели: тип «JavaScript-событие», идентификатор = название цели
3. UTM-метки собираются автоматически из URL и сохраняются с лидом

## Сбор лидов
Лиды сохраняются в localStorage. Для просмотра:
```javascript
JSON.parse(localStorage.getItem('fintablo_leads'))
```

### Интеграция с AmoCRM (подготовлено)
В файле `js/email.js` закомментирован блок для отправки в AmoCRM webhook.
Для подключения:
1. Создайте webhook или API-интеграцию в AmoCRM
2. Раскомментируйте блок и вставьте URL endpoint
3. Все поля квиза + UTM-метки передаются в теле запроса

## Режим отчёта
- **Полный отчёт** — после отправки контакта (все 7 разделов)
- **Превью** — без контакта (обзор + метрики + профиль, остальное заблокировано)
